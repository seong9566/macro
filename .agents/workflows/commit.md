---
description: 변경 사항을 분석하여 한국어 표준 커밋 메시지를 추천하고 커밋하는 워크플로우
---

1. 저장소 상태 확인: 먼저 `git status --short` 와 `git diff` (또는 `git diff --cached`)를 실행하여 현재 작업 중이거나 스테이징된 변경 사항을 파악합니다.
2. User scope (스코프) 결정: 변경된 파일이나 코드를 분석해 적절한 모듈 단위나 기능 단위의 스코프를 추론하거나, 사용자에게 어떤 스코프로 지정할지 물어봅니다.
3. 커밋 메시지 초안 생성: 아래의 커밋 타입(Type) 규칙에 맞춰 한글 커밋 메시지를 작성하고 사용자에게 제안합니다.
   - **양식**: `타입(User scope): 한글 요약 설명`
   - **허용되는 타입**:
     - `feat`: 새로운 기능 추가
     - `fix`: 버그 수정
     - `refactor`: 코드 리팩토링 (기능 변화가 없는 코드 구조 개선)
     - `docs`: 문서 추가 및 수정
     - `release`: 배포/릴리즈 관련 내용
4. 사용자 확인: "제안된 메시지들 중 어떤 것으로 커밋을 진행할까요?" 라고 확인을 받습니다.
5. 커밋 완료: 사용자가 승인한 커밋 메시지를 사용해 `git add`(선택)와 `git commit -m "승인된 메시지"`를 실행하여 커밋을 마칩니다.
