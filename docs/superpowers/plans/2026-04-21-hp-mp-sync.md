# HP/MP UI 동기화 구현 계획서

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 게임 화면의 캐릭터 HP/MP 바를 실시간으로 읽어 매크로 UI에 동기화 표시

**Architecture:** MacroEngine이 사냥 루프 내에서 HP/MP 비율을 측정하고, pyqtSignal 기반으로 UI 스레드에 전달. HP는 기존 HSV 색상 감지 방식 확장, MP는 파란색/보라색 HSV 범위로 새로 구현. UI에 MP 프로그레스바 추가.

**Tech Stack:** OpenCV (HSV 색상 감지), PyQt6 (QProgressBar, pyqtSignal), numpy

그리고 `_measure_player_hp()` 메서드 바로 뒤에 MP 측정 메서드를 추가한다. 동일하게 `frame` 파라미터를 받는다.

```python
def _measure_player_mp(self, frame=None):
    """
    캐릭터 MP바의 MP 비율(0.0~1.0)을 측정.
    게임 화면의 MP바 영역에서 파란색 픽셀 비율로 추정.

    Args:
        frame: 캡처된 프레임 (None이면 내부에서 캡처)

    Returns:
        float (0.0~1.0) 또는 -1.0 (측정 불가)
    """
    if frame is None:
        frame = capture_screen(region=self.region)
    if frame is None:
        return -1.0

    bx, by, bw, bh = PLAYER_MP_BAR_REGION
    if by + bh > frame.shape[0] or bx + bw > frame.shape[1]:
        return -1.0

    roi = frame[by:by + bh, bx:bx + bw]
    if roi.size == 0:
        return -1.0

    hsv = cv2.cvtColor(roi, cv2.COLOR_BGR2HSV)

    # 파란색~보라색 2구간 매칭 (그라데이션 대응)
    mask1 = cv2.inRange(hsv,
                        np.array(PLAYER_MP_COLOR_LOWER),
                        np.array(PLAYER_MP_COLOR_UPPER))
    mask2 = cv2.inRange(hsv,
                        np.array(PLAYER_MP_COLOR_LOWER2),
                        np.array(PLAYER_MP_COLOR_UPPER2))
    mask = cv2.bitwise_or(mask1, mask2)

    total_pixels = roi.shape[0] * roi.shape[1]
    if total_pixels == 0:
        return -1.0

    ratio = np.count_nonzero(mask) / total_pixels
    return ratio
```

## 참고 사항

### HP/MP 측정 방식
현재 HSV 색상 기반 픽셀 비율 방식을 사용한다. 이 방식의 한계:
- 정확한 수치(2853/3219)가 아닌 비율(%)로만 표시
- 바 영역의 정확한 좌표가 필요 (게임 해상도/위치 변경 시 재설정)

향후 OCR(Tesseract/EasyOCR)로 확장하면 정확한 수치 표시가 가능하지만, 현재는 비율 표시로 충분하며 성능 오버헤드가 없다.

### UI 스레드 안전성
`MacroEngine.player_hp_ratio`/`player_mp_ratio`는 float 값이므로 CPython GIL에 의해 원자적으로 읽기/쓰기된다. 별도 락이 필요 없다. 단, HP와 MP 대입 사이에 UI 스레드가 읽으면 "새 HP + 옛 MP" 조합이 보일 수 있으나, 개별 표시 구조이므로 시각적 문제는 없다.

### 기존 물약 시스템과의 관계
`_check_and_use_potion()`에서 `capture_screen()`을 1회만 호출하고, 그 프레임을 `_measure_player_hp(frame)`과 `_measure_player_mp(frame)`에 전달하여 추가 캡처 비용 없이 HP/MP를 함께 측정한다.

### UI 갱신 지연
`POTION_CHECK_INTERVAL = 1.0`초 + `stats_timer` 1초 간격이므로, HP/MP 바는 최대 1~2초 지연으로 표시된다. 실시간 사냥 매크로 용도에서는 충분한 수준이다.
